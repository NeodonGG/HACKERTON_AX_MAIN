**사용 방법**

```
python pipeline/generate_report.py <입력 CSV 경로> <출력 디렉터리>
```

- 원본 `data/voc.csv`(44건)로 검증: 수동 분류 대비 97.7%(43/44건) 일치
- 더미 `data/voc_dummy.csv`(200건)로 재검증: 코드 수정 없이 196건 처리(중복 4건 자동 제외), md/html 동일 품질로 생성 확인
- 매주 새 VoC CSV를 같은 컬럼 구조로 준비해 위 명령만 실행하면 정제→분류→집계→리포트가 자동 재현됨

**왜 이 리포트(2절 분류표)는 자동 분류 결과가 아닌가**: 파이프라인은 입력 CSV에 이미 검증된 `type` 컬럼이 있으면 규칙 기반 분류를 건너뛰고 그 값을 그대로 사용하도록 설계했다("정답 라벨이 있으면 신뢰하고, 없으면 예측한다"는 원칙). 이 리포트는 `output/voc-classification.csv`(2단계에서 사람이 직접 검증한 44건 분류)를 입력으로 생성했기 때문에 표에는 검증된 값이 실려 있다. 규칙 기반 분류기(정확도 97.7%, id 038 한 건만 불일치)는 정답 라벨이 없는 새 CSV — 위 검증에 쓴 원본·더미 데이터, 그리고 앞으로 들어올 실제 신규 VoC — 에 대해서만 실행되며, "파이프라인이 사람 개입 없이도 끝까지 재현되는가"를 증명하는 용도다. 즉 자동 분류와 수동 분류는 "더 정확한 쪽을 고르는" 관계가 아니라, 정답 유무에 따라 역할이 나뉜 관계다.

### 주간 VoC 리포트 설계

CS팀과 제품팀은 VoC 데이터에서 보고 싶은 정보가 다르다는 점을 반영해 섹션을 분리했다.
- CS팀: "지금 당장 대응해야 할 것"에 관심 → 최신 불만 건, 처리 지연 리스크
- 제품팀: "누적 데이터로 본 우선순위"에 관심 → 반복 기능요청, 규제 트렌드

**1. KPI 요약**
- 이번 주 총 접수 건수, 유형별 건수·비율, 전주 대비 증감률
- 근거: 유형 분포의 급변(예: 불만 급증)은 규제 일정·시스템 장애 등 즉시 조사 대상 신호이므로 최상단 배치

**2. 즉시 대응 목록 (CS팀 관점)**
- "불만" 유형 중 장애·마비·반려·긴급 등 고위험 키워드를 포함한 건을 우선순위로 나열
- 근거: CS팀은 개별 티켓 단위로 움직이므로, 분류 결과를 다시 개별 건 리스트로 풀어줘야 실무에 바로 쓸 수 있음

**3. 반복 기능 요청 TOP3 (제품팀 관점)**
- "기능 요청" 유형을 keyword_hint 기준으로 집계해 빈도 상위 3개 제시
- 근거: 제품 로드맵 의사결정은 개별 건이 아니라 반복되는 패턴의 빈도가 근거가 되어야 함

**4. 탄소규제 신규 패턴**
- CBAM·CSRD·Scope 등 규제 키워드 언급 빈도의 주간 추이와 규제 일정(신고기한 5/31, CBAM 인증서 구매 의무화 2026-01 등)의 연관성 표시
- 근거: industry-news.md에서 확인한 "신고기한 임박·규제기준 변경 시 VoC 급증" 패턴을 사전에 감지해 CS 인력 배치·콘텐츠 준비에 선제 대응하기 위함
